\subsection{[B2]}
Pour créer un tag, nous disposons de plusieurs paramètres.
On peut également créer un tag depuis bash directement en lui donnant un nom et un commit précis.
La version bash permet de faire un tag sur le commit que l'on veut alors que la version github nous bloque au commit les plus récent.
J'ai tout d'abord créé un premier tag avec le tout premier commit effectué sur le projet -> c'est la version 0.1.0.

Le tag v0.2.1 signifie qu'il y a eu une modification dans le document Latex mais pas d'ajout de nouvelles pages ou fonctionnalités

\section{Conclusion}

En conclusion, ce mini-projet nous a permis de mettre en pratique les notions abordées durant les séances de TD, notamment la gestion collaborative avec Git et GitHub. Nous avons pu constituer un groupe efficace et coordonner nos contributions grâce aux branches, aux \textit{pull requests} et à l’utilisation d’\textit{issues} pour le suivi des tâches. Le dépôt Git et les tags de version associé ont illustré de manière concrète la mise en place d’un projet structuré et collaboratif.

Malgré quelques difficultés techniques, notamment un problème de connexion rencontré par un membre du groupe qui a dû cloner le projet à plusieurs reprises, l’ensemble des objectifs a été atteint. Cette expérience nous a permis de mieux comprendre les bonnes pratiques de travail en équipe et de gestion de versions, tout en renforçant notre autonomie face aux outils professionnels de développement.
